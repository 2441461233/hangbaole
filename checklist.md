# 验收清单 (Checklist)

## UI & 动效 (Vibe)
- [ ] 初始输入框是否极简且居中？聚焦时是否有优雅的呼吸阴影？
- [ ] 点击发送后，输入框是否丝滑消失，流体加载动画是否自然平滑？
- [ ] “精神状态档案卡”弹出时，是否带有明显的物理阻尼感（弹簧动画）？
- [ ] 整体配色和排版是否符合“极简反差风”（苹果风的高级感 vs 发疯的文案）？

## 核心业务逻辑
- [ ] 用户输入内容后，API 能否稳定返回格式化 JSON 数据？
- [ ] `status` 字段能否正确显示大字的 `[夯爆了]` 或 `[拉完了]`？
- [ ] `quote` 字段是否充满反差感、反职场叙事的情绪价值？
- [ ] 雷达图（班味、发疯、离职、阴阳、干饭）是否根据用户输入动态变化且渲染正确？

## 社交传播能力
- [ ] 点击“保存/分享”按钮，是否能将档案卡完美保存为本地图片？
- [ ] 保存的图片中，毛玻璃背景、彩色阴影和中英文字体是否全部正确渲染？

## 工程化
- [ ] 在移动端（Mobile）浏览时，页面是否适配且没有横向滚动条？
- [ ] 出现网络错误或大模型抽风时，是否有优雅的报错提示，而不是白屏？
